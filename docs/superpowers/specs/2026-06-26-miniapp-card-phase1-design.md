# Перенос функций бота в Mini App — Этап 1: Карточка игрока

**Дата:** 2026-06-26
**Статус:** утверждён, готов к плану реализации

## Цель

Перенести функциональность из нижней reply-клавиатуры бота в Telegram Mini App
(`https://lflagent.ru`). Итоговая цель — клавиатура только из `[⚽ Открыть, 🆘 Помощь]`,
всё остальное — в Mini App. Делаем **поэтапно**:

- **Этап 1 (этот спек): Карточка игрока** — создать/редактировать/удалить/статус
- Этап 2: Команды (найти / зарегистрировать / моя команда)
- Этап 3: Избранное + финальная зачистка клавиатуры

Клавиатуру бота урезаем **в самом конце** (после Этапа 3). До этого бот работает как
раньше — страховка на время обкатки веб-версии.

## Объём Этапа 1

Полный цикл карточки игрока в Mini App, **без фото** (отложено отдельным шагом — требует
загрузки файла, хранения и показа; HTML-карточка пока с инициалами).

Поиск свободных агентов уже реализован (вкладка «Поиск») — не трогаем.

## Архитектура

Цепочка не меняется: `Mini App → https://lflagent.ru → Caddy → FastAPI (api.py) → SQLite`.
Добавляются новые эндпоинты в `api.py` и UI во вкладке «Карточка» (`webapp/index.html`).

Серверный парсер (`scraper/`, ходит на lfl.ru через РФ-прокси) переиспользуется как есть.
Запись в БД — через существующие функции `database/queries.py` (`upsert_agent`,
`deactivate_agent`, `update_looking`, `activate_agent`, `get_agent_by_tg_id`).

## Безопасность: проверка Telegram initData

**Проблема:** сейчас GET/POST используют непроверенный `tg_id` из query-параметра.
Для write-операций это дыра — любой мог бы изменить чужую карточку, зная `tg_id`.

**Решение:** на всех write-эндпоинтах проверять подпись Telegram WebApp `initData`:
1. Mini App шлёт строку `tg.initData` (в теле запроса или заголовке `X-Init-Data`).
2. Сервер парсит query-string, извлекает `hash`.
3. Строит `data_check_string` = отсортированные по ключу `key=value\n` (без `hash`).
4. `secret_key = HMAC_SHA256(key="WebAppData", msg=BOT_TOKEN)`.
5. `calc_hash = HMAC_SHA256(key=secret_key, msg=data_check_string)` (hex).
6. Сравнивает `calc_hash == hash` (constant-time). При несовпадении — `403`.
7. Берёт реальный `user.id` из поля `user` (JSON) — он и есть `tg_id` для записи.
8. Проверяет свежесть `auth_date` (не старше ~24ч) — отсечь переигрывание.

`BOT_TOKEN` берётся из env (`TELEGRAM_BOT_TOKEN`, уже в `.env`). Хелпер `verify_init_data(raw)`
→ возвращает `tg_id` или кидает `HTTPException(403)`.

GET-эндпоинты (`/api/me`, `/api/agents`) пока оставляем на `tg_id` (read-only, низкий риск).

## Новые API-эндпоинты (`api.py`)

Все write-эндпоинты принимают `init_data` (подписанная строка Telegram) и берут `tg_id` из неё.

### `POST /api/card/import`
Тело: `{ "init_data": "...", "url": "https://lfl.ru/..." }`
- Валидирует initData → `tg_id`.
- `detect_and_parse(url)` (как в боте). Ошибки парсера → `{ "ok": false, "error": "..." }` с 200/422.
- Сохраняет профиль: `upsert_agent(... active=0, profile_json=...)`.
- Ответ: `{ "ok": true, "profile": {...} }` (та же форма, что `/api/me`).

### `POST /api/card/save`
Тело: `{ "init_data", "name", "position", "age", "current_team", "division", "experience", "comment" }`
- Валидирует initData → `tg_id`. Серверная валидация: имя непустое; позиция ∈ {Нападающий,
  Полузащитник, Защитник, Вратарь}; возраст 10–60; лига ∈ {ЛФЛ, AFL, Pari Amateur, F-лига}.
- `upsert_agent(...)` (ручная карточка; `lfl_url=""`). Сохраняет `active`/`looking` без изменений,
  если запись уже была (редактирование).
- Ответ: `{ "ok": true, "profile": {...} }`.

### `POST /api/card/delete`
Тело: `{ "init_data" }` → `deactivate_agent(tg_id)` → `{ "ok": true }`.

### `POST /api/card/status`
Тело: `{ "init_data", "active"?: 0|1, "looking"?: 0|1 }`
- Выставляет переданные флаги (`activate_agent`/`deactivate_agent` для `active`,
  `update_looking` для `looking`).
- Ответ: `{ "ok": true, "profile": {...} }`.
- Заменяет текущий `POST /api/agent/toggle-free` (старый оставить как алиас или удалить —
  фронт переключаем на `/api/card/status`).

## Mini App — вкладка «Карточка» (`webapp/index.html`)

Состояния:

**Карточки нет** — экран выбора:
- Кнопка «🔗 Создать по ссылке» → поле ввода URL + «Загрузить» → спиннер → `POST /api/card/import`
  → показ полученной карточки → опционально доп. опыт/комментарий/статус → сохранение.
- Кнопка «✍️ Заполнить вручную» → форма (имя, позиция [дропдаун], возраст, текущая команда,
  лига [дропдаун], опыт, комментарий) → `POST /api/card/save`.

**Карточка есть** — текущий вид карточки + панель действий:
- «🔍 Ищу команду / ✋ Не ищу» (переключатель `looking`) → `POST /api/card/status`.
- «В поиск агентов / Убрать из поиска» (переключатель `active`) → `POST /api/card/status`.
- «✏️ Редактировать» → форма, предзаполненная текущими значениями → `POST /api/card/save`.
- «🗑 Удалить» → подтверждение → `POST /api/card/delete`.

UI на ванильном JS в стиле существующего `index.html` (без новых зависимостей).
Все запросы шлют `tg.initData`. Ошибки сервера показываем через `tg.showAlert`.

## Что остаётся в боте (Этап 1)

Все хендлеры карточки (`🃏 Создать карточку`, `🪪 Моя карточка` и т.д.) **остаются рабочими** —
дубль на время обкатки. Клавиатура без изменений. Зачистка — в Этапе 3.

## Тестирование

Pytest (как существующие тесты в `tests/`):
- `verify_init_data`: валидный/невалидный hash, протухший `auth_date`.
- `/api/card/save`: валидация полей (возраст, позиция, лига), создание и редактирование.
- `/api/card/import`: успех с моком `detect_and_parse`, ошибка парсера.
- `/api/card/delete`, `/api/card/status`: смена флагов.
